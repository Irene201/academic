Frågor

  1. <starttagg attributnamn='värde'> Innehåll </sluttagg>

    Element Taggarna och innehållet tillsammans kallas för element

    Attribut Attribut är ett nyckelord som står efter namnet i öppningstaggen och som anger att taggen ska ha en viss egenskap.

    Värde Värdet är 'innehållet' av attributet.

  2. HTTP betyder HyperText Transfer Protocol. Den används för alla vanliga webbsidor och för bilder, ljud och annat som ligger i webbsidor. HTTP är det kommunikationsprotokoll som används för att överföra webbsidor på informationsnätverket www på internet.

    Kommunikationsprotokoll är en överenskommelse mellan två eller flera parter om hur man ska kommunicera. I vardagslivet använder de flesta omedvetet kommunikationsprotokoll för att underlätta kommunikationen: när två personer möts säger båda 'hej' för att markera inledningen på samtal, och 'hej då' för att avsluta.

  3. Både HTML och CSS styrs av en organisation som heter W3C, World Wide Web Consortium, en jättelik sammanslutning med fler en 300 medlemsorganisationer. W3C utvecklar en mängd olika standarder för Internet, och när de är klara kallas de för rekommendationer.
  4. WWW står för World wide web. WWW är en internettjänst som gör webbplatser och andra datafiler åtkomliga via Internet eller Intranät. Webbsidorna är hypertextdokument som kan innehålla text, bilder, video och annan multimedia, samt klickbara hyperlänkar.

  5. En webbadress kallas en URL som betyder Uniform Resource Locator

    http:// kallas metod medan www.facebook.com kallas address och /home.php kallas sökväg.

  6. FTP används för att hänvisa till filer i filarkiv, och mailto som används för att skicka elektronisk post direkt från webbläsaren.
  7. HTTP bygger på ett förfrågan/svar-modell mellan client och server.
  8. HTML står för HyperText Markup Language och är ett sidbeskrivningsspråk som används för att skriva de webbsidor som utgör World Wide Web.
  9. Indentering används för att gruppera koden och skapa "stycken" som avskiljer koden i logiska block, men framförallt visa hur kodens strukturer förhåller sig till varandra hierarktiskt. På det viset kan programmeraren läsa koden lättare och hitta fel.
  10. Utseendet på sidan blir inte annorlunda oavsett om vi indenterar eller inte.
  11. Även om man använder WYSIWG så bör man ha grundkunskaper i hur HTML-språket är uppbyggt. Ofta måste man ändå kontrollera och korrigera HTML-koden manuellt. Kan man skriva egen kod så kan man göra så mycket mer än att använda ett program.
  12. head och body består hemsidan av. I head placeras metadata om vår webbplats, alltså information om vad som finns på webbsidan. I body placeras allt som visas för användaren av webbsidan.
  13. Det kan vara bra att kommentera så att man senare minns vad allt är till för. Just när vi lägger till något är det självklart, men hur mycket minns vi om fyra månader och fem andra koder senare? Eller om någon annan vill gå in och försöka förstå vad du gjort så är det enklare om det står förklarat.
